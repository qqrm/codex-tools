# Documentation and Contracts Review Skill

## What It Is
A reusable skill for verifying that published docs, examples, endpoint descriptions, and expected behavior stay synchronized.

## When To Use
- Public docs and implementation may have drifted.
- API or workflow changes need contract verification.
- The main risk is incorrect guidance for downstream users or agents.

## Default Persona
Use `analyst` for consumer-facing clarity and `quality_engineer` for consistency and acceptance criteria.
